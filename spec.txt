broker topics are of form
1. meshcore/iata/pubkey/status
2. meshcore/iata/pubkey/packets
and the data are in json format
IATA is a 3-letter upper case string
pubkey is a 64-character hexadecimal string

want to store the data in a sqlite database.
make it as fault tolerant as possible
if program crashes gracefully exit
optimize: there can be multiple concurrent reader processes (less than 10), message rate <10000/minute

status table:
name: origin string
pubkey: origin_id (primary key, unique)
radio: radio string
first_seen: UTC when first inserted into table
last_seen: UTC when updated if pubkey is already in the table

packets table:
timestamp: store UTC when message arrived (not from json)
packet_type: 1 byte number
route_type: from route: single letter can be 'F' or 'D'
iata: 3 letters in upper case
observer: 4 characters of pubkey in upper case
raw: hex string of length payload_len. each letter is a hex digit, so every 2 letters constitutes a hex byte
route_type: bits 0-1 of first byte of raw
payload_type: bits 2-5 of first byte of raw
version: bits 6-7 of first byte of raw

validation of packets (don't store if bad packet)
1) has raw data
2) length of raw string is double payload_len

put database code into a separate database.py file
